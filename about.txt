->Our minor project is a Question–Answer Retrieval System (like a mini Quora/StackOverflow search engine).

When a user types a new question such as:

Subject: “WiFi keeps dropping”
Body: “My laptop loses wireless connection every 5 minutes.”

your system should:

Understand the category of the question (Technology / Finance / Health / Education).

Search your dataset and find the most similar past questions.

Show the best matching answers (from those similar questions).

Use metadata (keywords/tags) to improve matching even when wording is different.

So the system is basically:
✅ “Predict category → enrich query with metadata/related words → retrieve similar questions → show answers”.

-> Why your project is good for a minor project

Because it includes multiple important AI/IR concepts:

Text classification (Transformer/BERT) to predict category.

Information retrieval (BM25) to search and rank.

Query expansion to handle different words meaning same thing.

Metadata usage (keywords/tags) to improve accuracy.

Evaluation using IR metrics like MAP and R-Precision.

Important terms explained (very simple)
1) Dataset

A collection of rows like:

qid, subject, body, category, answer, keywords, topic

We use this as “knowledge base”.

2) Category Classifier (Transformer/BERT / DistilBERT)

This model learns to classify text into categories:

Technology

Finance

Health

Education

Because the system can search smarter if it knows the domain.

Example:
If question is about UPI, searching Finance questions should be prioritized.

3) BM25 (search ranking algorithm)

BM25 is the “classic search engine ranking” method.

It gives a score based on:

how many query words appear in a document

how rare/important those words are

length normalization (long docs don’t win unfairly)

It’s fast and strong for text retrieval.

4) Metadata

Metadata = extra information about a question, like tags/keywords.

Example for WiFi:

wifi, router, driver, dns, network

Even if user doesn’t type “router”, metadata helps because:

similar questions having those keywords still match strongly.

5) Query Expansion

Means: adding extra related words to the user query to improve search.

Example:
“wifi dropping” can expand into:
“wireless network router driver disconnect”

This helps handle “lexical gap” (different words, same meaning).

6) Evaluation metrics

These measure retrieval quality:

MAP@K (Mean Average Precision)

Checks: are relevant results high in the ranking consistently?

R-Precision

Checks: if there are R relevant items, how many did you retrieve in top R?

MRR@K

Checks: how early does the first correct result appear?
